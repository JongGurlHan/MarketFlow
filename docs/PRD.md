# PRD: MarketStream — 실시간 시장 데이터 플랫폼

## 목표
여러 이종 공급처의 실시간 시세를 수집·정규화하여 하나의 표준 형식으로 REST/WebSocket API로 안정적으로 배포한다.

## 사용자
- **대외 API 소비자**: 표준화된 시세를 REST 스냅샷·WebSocket 스트림으로 소비하는 외부 시스템/개발자.
- **단말 사용자**: 실시간 대시보드(웹 단말)로 여러 심볼의 시세와 소스 상태를 모니터링하는 사용자.

## 핵심 기능
1. **수집**: 공급처의 실시간 피드(Upbit WebSocket)에 접속해 시세를 받는다. 공급처 추가는 `SourceConnector` 구현으로만.
2. **가공(정규화)**: 공급처마다 다른 원본 포맷을 단일 `NormalizedTick` 표준 스키마로 변환한다.
3. **처리·배포**: 정규화된 틱을 Redis pub/sub로 팬아웃하고 최신값을 캐시한다. REST(스냅샷)와 WebSocket(실시간 구독)으로 배포한다.
4. **단말**: 실시간 가격 그리드·소스 연결 상태를 보여주는 웹 대시보드.

## MVP 제외 사항 (phase 1 이후)
- 다중 공급처(Binance 등) — phase 1. 단, 인터페이스는 MVP에서 소스-무관하게 설계.
- 재접속·백오프·헬스 모니터링, API 키 인증·rate limit, OpenAPI 문서 — phase 1.
- 이력 저장(Elasticsearch), Prometheus/Grafana, 부하 테스트 지표 — phase 2.

## 도메인 프레이밍 ("왜 크립토냐")
금융정보 벤더 본업은 채권·외환·금리 등 전통 금융 시세다. 이 프로젝트가 Upbit(크립토)를 쓰는 이유는:
**자산군과 무관하게 실시간 틱을 수집·정규화·저지연 배포하는 기술적 난제는 동일**하며, 실 금융 데이터는
유료·폐쇄망인 반면 크립토는 동일 아키텍처를 검증할 수 있는 **공개 실시간 피드**를 제공하기 때문이다.
표준 스키마에 `assetClass` 필드를 두어 전통 금융 자산군으로의 확장을 구조적으로 대비한다.
