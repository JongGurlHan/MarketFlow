# UI 디자인 가이드

대상: `apps/web` 실시간 대시보드(웹 단말).

## 디자인 원칙
1. **도구처럼 보여야 한다.** 마케팅 랜딩이 아니라 매일 들여다보는 시세 모니터링 단말.
2. **정보 밀도 우선.** 여백보다 한 화면에 담기는 심볼·수치의 양이 중요. 숫자는 모노스페이스로 정렬.
3. **시세 변화가 즉시 읽혀야 한다.** 상승/하락/소스 상태를 색으로 즉각 구분.

## AI 슬롭 안티패턴 — 하지 마라
| 금지 사항 | 이유 |
|-----------|------|
| backdrop-filter: blur() | glass morphism은 AI 템플릿의 가장 흔한 징후 |
| gradient-text (배경 그라데이션 텍스트) | AI가 만든 SaaS 랜딩의 1번 특징 |
| "Powered by AI" 배지 | 기능이 아니라 장식 |
| box-shadow 글로우 애니메이션 | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 rounded-2xl | 균일한 둥근 모서리는 템플릿 느낌 |
| 배경 gradient orb (blur-3xl 원형) | 모든 AI 랜딩 페이지에 있는 장식 |

## 컴포넌트
### 시세 테이블 (핵심)
- 열: 심볼 · 현재가 · 24h 변동률 · 소스 · 최종 갱신. 숫자는 우측 정렬, `font-mono`, tabular-nums.
- 현재가 갱신 시 상승은 초록/하락은 빨강으로 셀을 짧게(150ms) 플래시 후 원복. 그 외 애니메이션 금지.

### 카드/패널
```
rounded-md bg-[#141414] border border-neutral-800 p-4
```

### 상태 인디케이터
- 소스 연결 상태는 작은 원형 점(초록/노랑/빨강) + 레이블. 글로우·펄스 애니메이션 금지(단순 색상만).

## 레이아웃
- 전체 너비: max-w-6xl, 좌측 정렬 기본. 중앙 정렬 금지.
- 상단: 제목 + 소스 상태 바. 본문: 시세 테이블. 간격 gap-3~4, 섹션 간 space-y-6.

## 타이포그래피
| 용도 | 스타일 |
|------|--------|
| 페이지 제목 | text-2xl font-semibold text-neutral-100 |
| 패널 제목 | text-sm font-medium text-neutral-400 |
| 시세 숫자 | text-sm font-mono tabular-nums text-neutral-100 |

## 애니메이션
- 가격 변경 플래시(150ms)만 허용. 그 외 모든 애니메이션 금지.

## 아이콘
- SVG 인라인, strokeWidth 1.5. 아이콘을 둥근 배경 박스로 감싸지 않는다.
